    1. **Delete `apps/api/nodemon.json`** -- no longer needed.

    2. **Update `apps/api/nest-cli.json`** to enable SWC compilation:
    ```json
    {
      "$schema": "https://json.schemastore.org/nest-cli",
      "collection": "@nestjs/schematics",
      "sourceRoot": "src",
      "compilerOptions": {
        "builder": "swc",
        "deleteOutDir": true
      }
    }
    ```
    The `"builder": "swc"` makes `nest build` use SWC instead of tsc, and `nest build --watch` uses SWC for fast incremental rebuilds.

    3. **Update `apps/api/package.json`** dev script:
    ```json
    { "dev": "nest build --watch" }
    ```
    Remove `nodemon` from devDependencies. Also remove `detect-port` from dependencies and `@types/detect-port` from devDependencies (unused, per research).

    4. **Update `apps/web/package.json`** -- rename the `dev` script to `dev:standalone` so turbo does NOT pick it up as a dev task. The web app is served via embedded Vite middleware in the API's main.ts during dev:
    ```json
    {
      "scripts": {
        "dev:standalone": "vite",
        "build": "tsc -b && vite build",
        ...rest unchanged
      }
    }
    ```

    5. **Update `turbo.json`** dev task to be dependency-aware:
    ```json
    {
      "tasks": {
        "dev": {
          "dependsOn": ["^dev"],
          "cache": false,
          "persistent": true
        },
        ...rest unchanged
      }
    }
    ```
    This ensures library dev tasks (tsc --watch) start before the API dev task (nest build --watch).

    6. **Update root `package.json`** dev script:
    ```json
    {
      "scripts": {
        "dev": "turbo watch dev",
        "dev:all": "turbo watch dev --concurrency 20",
        ...rest unchanged
      }
    }
    ```
    `turbo watch dev` replaces the two-step `turbo build && turbo dev`. Turbo watch monitors file changes and re-runs dependent tasks automatically. Since the web package no longer has a `dev` script, it will not be included.

    **Why this works:**
    - `turbo watch dev` starts all packages that have a `dev` script
    - Libraries (shared, connector-sdk, connectors) run `tsc --watch` (their existing dev scripts)
    - API runs `nest build --watch` (SWC-backed, auto-restarts on source changes)
    - Web has no `dev` script, so turbo skips it -- frontend is served by embedded Vite in API's main.ts
    - `dependsOn: ["^dev"]` ensures library builds complete before API starts
    - New connectors with a `dev` script are automatically discovered by turbo -- zero root config changes needed (DEV-03)
