Create a standalone one-time migration script at `apps/api/src/migrations/backfill-entity-types.ts` that can be run with `npx tsx apps/api/src/migrations/backfill-entity-types.ts`.

The script should:

1. Open the SQLite database at `process.env.DB_PATH || './data/botmem.db'` using `better-sqlite3` directly (not Drizzle -- this is a one-time script).

2. **Normalize memory entities:**
   - SELECT all rows from `memories` where `entities` is not null and not empty ('[]').
   - For each row, parse the entities JSON array.
   - Filter out entities with type in ('time', 'amount', 'metric') -- delete them entirely.
   - For remaining entities, map each to `{type, value}` only (strip `confidence` and any other fields).
   - Normalize type values: map any non-canonical types to 'other'. Canonical set: person, organization, location, event, product, topic, pet, group, device, other.
   - UPDATE the row with the cleaned JSON.
   - Use a transaction for the full batch.

3. **Normalize contact entityType:**
   - UPDATE contacts SET entity_type = 'other' WHERE entity_type NOT IN ('person', 'organization', 'location', 'event', 'product', 'topic', 'pet', 'group', 'device', 'other').
   - This handles the expansion from person|group|organization|device to the full canonical set (group and device are already in the canonical set, so only truly unknown values get mapped to 'other').

4. Run silently -- no logging of counts/changes per the user decision. Just a final "Migration complete" message.

5. Update the schema comment on `contacts.entityType` in `apps/api/src/db/schema.ts` from `// person | group | organization | device` to `// person | organization | location | event | product | topic | pet | group | device | other`.
