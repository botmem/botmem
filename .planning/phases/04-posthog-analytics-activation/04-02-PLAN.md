<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create PostHog project and provide API keys</name>
  <action>
User must create a PostHog Cloud project and provide the API key.

Steps:
1. Go to https://us.posthog.com and sign in (or create account)
2. Create a new project (or use existing one)
3. Go to Project Settings -> Project API Key
4. Copy the API key

Then set the environment variables. Add to your shell profile or a `.env` file that your dev setup reads:
```
POSTHOG_API_KEY=phc_your_key_here
VITE_POSTHOG_API_KEY=phc_your_key_here
```

For the Vite frontend, the key must be available at build time. If using `.env` files, create `apps/web/.env.local`:
```
VITE_POSTHOG_API_KEY=phc_your_key_here
```

And for the API, either export in shell or create `apps/api/.env`:
```
POSTHOG_API_KEY=phc_your_key_here
```

Restart both dev servers after setting the keys.
  </action>
  <resume-signal>Provide the PostHog API key or confirm keys are set and servers restarted</resume-signal>
</task>
