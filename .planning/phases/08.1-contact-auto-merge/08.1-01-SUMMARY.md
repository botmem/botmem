# Phase 8.1 Plan 01: Contact Entity Type Reclassification Summary

**Reclassify contact entityTypes by cross-referencing memory entity data, with strict thresholds to avoid false positives**

## Accomplishments
- Added reclassifyEntityTypes() method that cross-references memory entity data to identify non-person contacts
- 48 contacts reclassified from 'person' to correct types (organization, location, product, event) on first run
- Known targets (Apple, Google Cloud, FAB, RTA, noon, Wio, Justlife, Mashreq, OpenAI, GitHub, etc.) correctly reclassified
- resolveContact now preserves non-person entityType on existing contacts during future syncs
- Operation is fully idempotent (second run returns reclassified: 0)

- `apps/api/src/contacts/contacts.controller.ts` - Added POST /people/reclassify endpoint

## Decisions Made
- Used strict 3x threshold (non-person count must be >= 3x person count) to minimize false positives -- entity extraction data is noisy and person names often appear as locations/organizations
- Skip contacts with phone-number or Slack-ID display names since they can't be meaningfully reclassified
- Added typeof string guard on entity.value since some entities have numeric values that cause runtime TypeError

## Deviations from Plan

- **Files modified:** apps/api/src/contacts/contacts.service.ts
- **Verification:** Endpoint returns results without errors
- **Committed in:** 98fa0a1 (Task 1 commit)

**2. [Rule 1 - Bug] Fixed route ordering for POST /people/reclassify**
- **Found during:** Task 1 (endpoint verification)
- **Issue:** Route placed after :id routes could cause matching conflicts in NestJS
- **Fix:** Moved @Post('reclassify') before @Get(':id') in controller
- **Files modified:** apps/api/src/contacts/contacts.controller.ts
- **Committed in:** 98fa0a1 (Task 1 commit)

**3. [Rule 1 - Bug] Strengthened reclassification threshold to avoid false positives**
- **Found during:** Task 1 (verification)
- **Issue:** Initial algorithm (any 1+ non-person match) reclassified 132 contacts including many real persons
- **Fix:** Added strict 3x threshold, phone/ID filtering, and minimum 2-occurrence requirement
- **Files modified:** apps/api/src/contacts/contacts.service.ts
- **Committed in:** 98fa0a1 (Task 1 commit)
