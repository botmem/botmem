**Step A: Add `setPayload` method to QdrantService.**

In `apps/api/src/memory/qdrant.service.ts`, add a new public method:

```typescript
async setPayload(
  payload: Record<string, unknown>,
  filter: Record<string, unknown>,
): Promise<void> {
  await this.client.setPayload(QdrantService.COLLECTION, {
    payload,
    filter,
    wait: true,
  });
}
```

This wraps the Qdrant client's `setPayload` for batch payload updates without touching vectors.

**Step B: Create standalone migration script.**

Create `apps/api/src/migrations/backfill-source-types.ts` following the pattern from `backfill-entity-types.ts`. The script:

1. Connects directly to SQLite via `better-sqlite3` (not NestJS DI) using `DB_PATH` env var (default `./data/botmem.db`).
2. Connects to Qdrant via `QdrantClient` using `QDRANT_URL` env var (default `http://localhost:6333`).
3. Prints before-counts: `SELECT source_type, COUNT(*) FROM memories WHERE connector_type = 'photos' GROUP BY source_type`.
4. In a single SQLite transaction:
   - `UPDATE memories SET source_type = 'photo' WHERE connector_type = 'photos' AND source_type = 'file'` -- log changes count.
   - `UPDATE raw_events SET source_type = 'photo' WHERE connector_type = 'photos' AND source_type = 'file'` -- log changes count.
5. Qdrant batch update: `client.setPayload('memories', { payload: { source_type: 'photo' }, filter: { must: [{ key: 'source_type', match: { value: 'file' } }, { key: 'connector_type', match: { value: 'photos' } }] }, wait: true })`.
6. Prints after-counts for verification.
7. Closes DB connection.

CRITICAL: Always filter by BOTH `source_type = 'file'` AND `connector_type = 'photos'` to avoid affecting Slack file attachments.

The script must be runnable with `npx tsx apps/api/src/migrations/backfill-source-types.ts`.
