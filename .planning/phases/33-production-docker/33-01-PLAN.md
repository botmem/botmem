<task type="auto">
  <name>Task 1: Remove @botmem/web dependency and guard ServeStaticModule</name>
  <files>apps/api/package.json, apps/api/src/app.module.ts</files>
  <action>
1. In `apps/api/package.json`, remove `"@botmem/web": "workspace:*"` from dependencies. This prevents turbo prune from including the entire web app in the API-only Docker image. The API never imports from @botmem/web -- ServeStaticModule uses a filesystem path.

2. In `apps/api/src/app.module.ts`, add `import { existsSync } from 'fs';` at the top. Change the ServeStaticModule conditional from:

```typescript
const isDev = process.env.NODE_ENV !== 'production';
// and the spread:
...(isDev ? [] : [ServeStaticModule.forRoot({...})])
```

to:

```typescript
const webDistPath = join(__dirname, '..', '..', 'web', 'dist');
const serveStatic = !isDev && existsSync(webDistPath);
// and the spread:
...(serveStatic ? [ServeStaticModule.forRoot({
  rootPath: webDistPath,
  exclude: ['/api/{*path}'],
})] : [])
```

This ensures the API starts cleanly whether web/dist exists or not (Docker API-only image vs full deployment).

3. Run `pnpm install` to update the lockfile after removing the dependency.

<task type="auto">
  <name>Task 2: Create Dockerfile, .dockerignore, and .npmrc for production build</name>
  <files>Dockerfile, .dockerignore, .npmrc</files>
  <action>
1. Create `.npmrc` at project root with:
```
shamefully-hoist=true
```
This is required for NestJS to resolve dependencies in Docker's pnpm install. Note: this file may affect local dev behavior -- pnpm already works locally without it because NestJS finds deps via the workspace symlink structure. In Docker, without workspace context, hoisting is needed.

2. Create `.dockerignore` at project root:

```
node_modules
.git
.turbo
dist
coverage
data
*.log
.env*
!.env.example
.DS_Store

apps/web/dist
convo.md
```

3. Create `Dockerfile` at project root with a 4-stage multi-stage build:

**Stage 1 - base:** `node:22-alpine`, enable corepack, prepare `pnpm@9.15.0`, install turbo globally.

**Stage 2 - pruner:** Copy full repo, run `turbo prune @botmem/api --docker`. This generates `out/json/` (package manifests + pruned lockfile) and `out/full/` (source code for API + its workspace deps only).

**Stage 3 - builder:** Install build tools for native modules (`python3 make g++`). Copy `.npmrc`. Copy `out/json/` first and run `pnpm install --frozen-lockfile` (layer cached on lockfile changes only). Then copy `out/full/` and run `pnpm turbo build --filter=@botmem/api`. Finally run `pnpm install --prod --frozen-lockfile` to prune dev dependencies.

**Stage 4 - runner:** `node:22-alpine`, create non-root user `nestjs` (uid 1001), copy `dist/` from builder at `apps/api/dist`, copy `package.json` from builder at `apps/api/package.json`, copy root `node_modules/` (hoisted, prod-only). Create `/app/data` directory owned by nestjs for SQLite. Set `NODE_ENV=production`, `PORT=12412`, expose 12412. CMD `["node", "dist/main.js"]`.

Key details:

- Pin pnpm version to match `packageManager` field exactly: `9.15.0`
- Pin turbo to `^2.4.0` to match project version
- Use `apk add --no-cache` for build tools (only in builder stage)
- Non-root user in production stage for security
- The `--frozen-lockfile` flag ensures reproducible installs
