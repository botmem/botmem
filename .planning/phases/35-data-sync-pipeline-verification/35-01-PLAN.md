<task type="auto">
  <name>Task 2: Create pipeline verification script</name>
  <files>scripts/verify-pipeline.ts</files>
  <action>
Create `scripts/verify-pipeline.ts` — a standalone script that validates pipeline correctness for a given connector type (or all). Run via `npx tsx scripts/verify-pipeline.ts [connector-type]`.

The script should connect to PostgreSQL and Qdrant directly (read DATABASE_URL and QDRANT_URL from env or defaults).

Checks per connector type:

1. **Raw events exist**: `SELECT COUNT(*) FROM raw_events WHERE connector_type = ?` — must be > 0
2. **Memories created**: `SELECT COUNT(*) FROM memories WHERE connector_type = ?` — must be > 0
3. **Correct source_type**: Verify memories have expected source_type per connector:
   - gmail → email
   - slack → message
   - whatsapp → message
   - imessage → message
   - photos-immich → photo
   - owntracks → location
4. **Pipeline complete**: `SELECT COUNT(*) FROM memories WHERE connector_type = ? AND pipeline_complete = true` — should equal total memories count
5. **Embeddings exist**: Check memories have non-null embedding (or check Qdrant point count matches memory count for that connector)
6. **Contacts resolved**: `SELECT COUNT(DISTINCT contact_id) FROM memory_contacts mc JOIN memories m ON mc.memory_id = m.id WHERE m.connector_type = ?` — must be > 0 for email/message types
7. **Entities extracted**: Check memories have non-empty entities JSON array
8. **Factuality set**: Check memories have non-null factuality_label
9. **Qdrant vectors**: Query Qdrant to count points with filter `connector_type = ?` — must match memory count
10. **Search sanity**: Use Qdrant to do a simple vector search filtered by connector_type and verify results come back

Output format: table with connector | check | status (PASS/FAIL) | details.
Support `--connector <type>` flag to check one connector, or no flag to check all.

Use `pg` library directly (not Drizzle) to keep the script standalone. Use `@qdrant/js-client-rest` for Qdrant checks.
</action>
