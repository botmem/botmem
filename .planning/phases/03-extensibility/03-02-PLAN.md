  <action>
    NOTE: PluginsModule is @Global() so MemoryModule does NOT need to import it. Just inject PluginRegistry directly.

    1. **memory.processor.ts** -- Inject PluginRegistry via constructor. Add three fire-and-forget hook calls:
       - After the memory record is inserted into DB (after the INSERT/upsert step): `void this.pluginRegistry.fireHook('afterIngest', { id: memoryId, text, sourceType, connectorType, eventTime })`
       - After Qdrant upsert completes: `void this.pluginRegistry.fireHook('afterEmbed', { id: memoryId, text, sourceType, connectorType, eventTime })`
       - After enrichService.enrich() completes: `void this.pluginRegistry.fireHook('afterEnrich', { id: memoryId, text, sourceType, connectorType, eventTime, entities: memory.entities, factuality: memory.factuality })`
       - All calls use `void` (fire-and-forget, not awaited). The `void` keyword explicitly signals intentional non-awaiting.

    2. **embed.processor.ts** -- Inject PluginRegistry via constructor. Add hook calls at the same logical points:
       - After DB insert: `void this.pluginRegistry.fireHook('afterIngest', ...)`
       - After Qdrant upsert: `void this.pluginRegistry.fireHook('afterEmbed', ...)`

    3. **enrich.processor.ts** -- Inject PluginRegistry via constructor. Add hook call:
       - After enrichment completes: `void this.pluginRegistry.fireHook('afterEnrich', ...)`

    4. **memory.service.ts** -- Inject PluginRegistry via constructor. Two changes:
       a. In the `search()` method, after computing final results but before returning: `void this.pluginRegistry.fireHook('afterSearch', { query, resultCount: results.length, topScore: results[0]?.score })` -- fire-and-forget, does NOT delay the response.
       b. In `computeWeights()`: After computing the existing final score, check for scorer plugins:
          ```
          const scorers = this.pluginRegistry.getScorers();
          if (scorers.length > 0) {
            let pluginBonus = 0;
            for (const scorer of scorers) {
              try { pluginBonus += scorer.score(mem, weights); } catch { /* ignore */ }
            }
            pluginBonus = Math.max(-0.05, Math.min(0.05, pluginBonus / scorers.length));
            final = Math.max(0, Math.min(1, final + pluginBonus));
          }
          ```
          Scorer bonus is averaged across plugins, clamped to +/-0.05, and the final score is clamped to [0,1].

    5. **memory.module.ts** -- No import changes needed (PluginsModule is @Global). But if PluginRegistry needs explicit provider listing for injection, it should already be available globally. No changes needed here unless testing reveals otherwise.

    IMPORTANT: Use `void` prefix on all fireHook calls in the pipeline hot path. This makes intent explicit (we are deliberately not awaiting) and satisfies TypeScript's no-floating-promises lint rule if enabled.
