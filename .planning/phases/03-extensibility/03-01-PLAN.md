<task type="auto" tdd="true">
  <name>Task 1: Create plugin types and PluginRegistry class with tests</name>
  <files>apps/api/src/plugins/plugin.types.ts, apps/api/src/plugins/plugin-registry.ts, apps/api/src/plugins/__tests__/plugin-registry.test.ts</files>
  <behavior>
    - registerLifecycle stores a lifecycle plugin and its hooks are callable via fireHook
    - registerScorer stores a scorer plugin retrievable via getScorers
    - fireHook('afterEnrich', data) calls all lifecycle plugins that registered an afterEnrich handler
    - fireHook passes a frozen shallow copy of the data (Object.freeze({...data})) so plugins cannot mutate pipeline state
    - A hook handler that throws is caught and logged, other handlers still execute (Promise.allSettled)
    - fireHook with no registered handlers for that hook completes without error
    - getScorers returns a copy of the scorer array
  </behavior>
  <action>
    1. Create `apps/api/src/plugins/plugin.types.ts` with these interfaces:
       - `PluginManifest`: { name: string, version: string, type: 'connector' | 'scorer' | 'lifecycle', description?: string, hooks?: string[], entryPoint?: string }
       - `HookName`: union type 'afterIngest' | 'afterEmbed' | 'afterEnrich' | 'afterSearch'
       - `LifecyclePlugin`: { manifest: PluginManifest, hooks: Partial<Record<HookName, (memory: Record<string, unknown>) => void | Promise<void>>> }
       - `ScorerPlugin`: { manifest: PluginManifest, score: (memory: Record<string, unknown>, currentWeights: Record<string, number>) => number }

    2. Create `apps/api/src/plugins/plugin-registry.ts`:
       - `PluginRegistry` class (Injectable)
       - Private arrays: lifecyclePlugins, scorerPlugins
       - Logger via NestJS Logger
       - `registerLifecycle(plugin)`: pushes to lifecyclePlugins array
       - `registerScorer(plugin)`: pushes to scorerPlugins array
       - `async fireHook(hook: HookName, data: Record<string, unknown>)`: filters lifecycle plugins that have the hook, maps to handler calls with Object.freeze({...data}), uses Promise.allSettled, logs any rejections as warnings
       - `getScorers()`: returns shallow copy of scorerPlugins array
       - `getLifecyclePlugins()`: returns shallow copy (for debugging/listing)

    3. Create `apps/api/src/plugins/__tests__/plugin-registry.test.ts` with tests covering all behaviors listed above. Use vi.fn() for mock handlers. Test that a throwing handler does not prevent other handlers from executing. Test that frozen data cannot be mutated by a handler.
  </action>

  </verify>

    - A subdirectory with manifest.json type=connector is skipped (handled by existing ConnectorRegistry)
    - A subdirectory with invalid/missing manifest.json logs a warning and continues

    - PluginsModule is @Global() and exports PluginRegistry for injection

         f. If type=scorer: dynamic import, check for score function, call registry.registerScorer
